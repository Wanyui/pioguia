# Security Policy

## Reportar una vulnerabilidad

Si detectas un problema de seguridad, no abras una issue publica.

Comparte el reporte de forma privada con:

- descripcion del problema
- pasos para reproducir
- impacto potencial
- propuesta de mitigacion (si aplica)

## Alcance

Se priorizan reportes relacionados con:

- inyeccion de contenido (XSS)
- exposicion de datos sensibles
- configuraciones inseguras de despliegue
- dependencias con vulnerabilidades criticas

## Tiempo de respuesta

Objetivo orientativo:

- acuse de recibo: 72 horas
- primera evaluacion: 7 dias

## Coordinacion de divulgacion

Se agradece divulgacion responsable: esperar confirmacion y correccion antes de publicar detalles tecnicos.
